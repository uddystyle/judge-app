### 実装の経緯

#### 当初の問題: Vitestのモックホイスティング

最初の実装試行では、以下の問題に直面しました:

```typescript
// ❌ これは動かない
let mockGoto: any;
vi.mock('$app/navigation', () => ({
  goto: (...args) => mockGoto(...args) // mockGotoはundefined
}));

beforeEach(() => {
  mockGoto = vi.fn(); // ← vi.mock()より後に実行される
});
```

`vi.mock()`はファイルの先頭にホイスティングされますが、`beforeEach`はテスト実行時に初めて実行されます。そのため、モック内で`beforeEach`で初期化される変数を参照できませんでした。

#### ✅ 解決策: グローバルモック変数

最終的に、以下のアプローチで実装に成功しました:

```typescript
// ✅ これは動く
const mockGoto = vi.fn(); // グローバルスコープで宣言

vi.mock('$app/navigation', () => ({
  goto: mockGoto // ホイスティング時点で参照可能
}));

beforeEach(() => {
  vi.clearAllMocks(); // クリアのみ行う
  // 必要に応じて再設定
});
```

**キーポイント**:
1. モック変数をグローバルスコープで宣言
2. `vi.mock()`でグローバル変数を参照
3. `beforeEach`ではクリアと再設定のみ
4. チェーン可能なモック（`channel().on().subscribe()`）を実装

### 検証内容

実際に`+page.svelte`をマウントして、以下を検証:

- ✅ `supabase.channel(channelName)`が呼ばれる
- ✅ `channel.on('postgres_changes', {...}, callback)`が正しい設定で呼ばれる
- ✅ `channel.subscribe(statusCallback)`が呼ばれる
- ✅ 主任検定員の場合は購読がスキップされる
- ✅ アンマウント時に`supabase.removeChannel()`が呼ばれる

### 以前試して失敗した解決策

1. **beforeEachでのモック初期化** → ホイスティング問題で失敗
2. **vi.doMock()** → SvelteKitのビルドシステムと競合
3. **resolve条件の追加** → モック問題は解決せず
